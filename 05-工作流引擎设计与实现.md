# 工作流引擎设计与实现

## 1. 流程设计器
- `workflow/designer-schema.json` 配置化定义节点类型与属性面板字段。

## 2. 流程定义
- `WorkflowDefinition`、`NodeDefinition`、`TransitionDefinition` 支持版本化与条件流转。

## 3. 节点引擎
- `NodeEngine` 负责生成审批任务，并根据 `signType` 实现单人审批/会签(AND)/或签(OR)。

## 4. 流程执行器
- `WorkflowRuntimeService` 提供 deploy/start/approve/reject/recall。
- 支持流程变量、条件表达式匹配下一节点。

## 5. 审批记录
- `auditLogs` 记录 APPROVE/REJECT/RECALL 轨迹，可扩展到数据库持久化。

## 6. 流程状态机
- `ProcessStateMachine` 控制 DRAFT/RUNNING/APPROVED/REJECTED/CANCELED 合法迁移。
