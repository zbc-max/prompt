# 企业级低代码开发平台（仅系统架构版）

> 按要求本稿**只输出系统架构**，不包含 ER、数据库 SQL、详细表结构。

## 1. 总体架构目标

- 元数据驱动（Metadata Driven）
- 微内核 + 插件化（Microkernel + Plugin）
- 前后端分离（Vue3 + Spring Boot）
- 多租户 SaaS 可演进
- 高内聚低耦合、支持热插拔扩展

---

## 2. 整体技术架构

### 2.1 分层架构

```text
┌──────────────────────────────────────────────┐
│                 展示层（Vue3）               │
│  设计器端 / 运行时端 / 管理后台 / 大屏端      │
└──────────────────────────────────────────────┘
                     │ HTTP/WebSocket
┌──────────────────────────────────────────────┐
│               接入层（Nginx）                │
│  静态资源、反向代理、负载均衡、CORS、限流     │
└──────────────────────────────────────────────┘
                     │
┌──────────────────────────────────────────────┐
│            应用层（Spring Boot）             │
│  统一API / Sa-Token认证 / 租户上下文 / 审计   │
└──────────────────────────────────────────────┘
                     │
┌──────────────────────────────────────────────┐
│         微内核层（Kernel + SPI）             │
│  权限核心 / 元数据核心 / 插件核心 / API核心   │
└──────────────────────────────────────────────┘
                     │
┌──────────────────────────────────────────────┐
│          引擎层（可插件化能力）              │
│  表单/页面/流程/规则/报表/代码生成等引擎       │
└──────────────────────────────────────────────┘
                     │
┌──────────────────────────────────────────────┐
│      基础设施层（MySQL/Redis/MinIO/Quartz）  │
│         持久化、缓存、对象存储、任务调度       │
└──────────────────────────────────────────────┘
```

### 2.2 运行架构（容器化）

- `nginx`：前端静态站点 + 网关入口
- `lowcode-server`：Spring Boot 单体（后续可拆分微服务）
- `mysql8`：核心业务库
- `redis`：缓存/会话/分布式锁
- `minio`：附件与模板资源
- `quartz`：定时任务执行器（与服务同进程或独立调度节点）

## 3. 微内核架构设计

### 3.1 内核职责边界

1. **tenant-kernel（租户内核）**
   - 统一租户上下文解析
   - 统一租户数据隔离策略注入

2. **auth-kernel（认证内核）**
   - 基于 Sa-Token 登录态管理
   - Token 生命周期、续签、踢下线

3. **rbac-kernel（权限内核）**
   - 用户/角色/菜单/按钮授权模型
   - 数据权限与字段权限判定接口

4. **metadata-kernel（元数据内核）**
   - Schema 存储、版本、发布、回滚
   - 统一元数据读取 API

5. **plugin-kernel（插件内核）**
   - 插件安装、启停、卸载生命周期
   - SPI 扩展点注册与冲突检测

6. **openapi-kernel（开放接口内核）**
   - API Key、签名验证、限流、审计

### 3.2 插件机制（热插拔）

插件必须声明：

- 插件元信息（名称、版本、兼容内核版本）
- 依赖关系（依赖哪些核心能力或其他插件）
- 扩展点实现（字段组件、流程处理器、规则函数等）
- 路由、菜单、权限点注册信息

插件生命周期：

`install -> enable -> disable -> uninstall`

## 4. 核心模块划分（系统视角）

### 4.1 设计态模块（Design Time）

- 表单设计器（Schema Designer）
- 页面设计器（Page Designer）
- 流程设计器（Workflow Designer）
- 报表与大屏设计器（Report/Screen Designer）

### 4.2 运行态模块（Runtime）

- 动态渲染引擎（Render Runtime）
- 流程执行引擎（Workflow Runtime）
- 规则执行引擎（Rule Runtime）
- 报表执行引擎（Report Runtime）

### 4.3 平台治理模块（Governance）

- 租户管理
- 用户与权限中心
- 操作审计与日志中心
- 系统监控与告警
- 配置中心（可对接 Nacos）

## 5. 元数据驱动主流程

### 5.1 设计发布链路

1. 设计器生成 JSON Schema / DSL
2. 元数据内核保存草稿版本
3. 发布时生成“可运行快照版本”
4. 权限内核绑定菜单与操作权限
5. 运行态按版本加载并渲染执行

### 5.2 运行执行链路

1. 用户访问页面
2. 鉴权 + 租户上下文注入
3. 读取已发布元数据版本
4. 渲染 UI + 执行联动/规则/流程
5. 数据写入业务存储并记录审计

---

## 6. 扩展性设计要点

- **扩展点优先**：所有引擎能力通过 SPI 暴露扩展点。
- **配置优先**：业务变化尽量通过元数据变化完成。
- **版本优先**：所有核心配置可回滚、可灰度。
- **隔离优先**：租户、权限、插件三重隔离。
- **审计优先**：设计变更与运行操作全链路可追踪。

## 7. 演进路线（架构层）

### 阶段 A：单体增强
- Spring Boot 单体 + 微内核 + 插件机制
- 快速交付核心低代码能力

### 阶段 B：服务拆分
- 按能力拆分：认证中心、元数据中心、流程中心、报表中心
- 引入消息总线解耦跨域调用

### 阶段 C：SaaS 强化
- 租户资源配额
- 租户级插件市场
- 多环境发布（开发/测试/生产）
